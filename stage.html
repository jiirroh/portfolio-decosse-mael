Stage de Deuxième Année : "theALaMenthe"

Application Web - theALaMenthe

Application de gestion pour un établissement (restaurant/café) avec système de commande (carte), authentification et panneau d'administration.

1. Contexte & Objectifs

Ce projet a été réalisé lors de mon stage de deuxième année. L'objectif était de développer une application web dynamique et sécurisée permettant de gérer une carte de produits et d'offrir une interface d'administration.
Environnement : Serveur local (UwAmp), développement Full-Stack.

2. Architecture Technique

  • Langages : PHP (Logique côté serveur), HTML/CSS (Interfaces), JavaScript.
  • Architecture : Modèle-Vue-Contrôleur (MVC) pour une séparation propre de la logique métier (modèles), de l'affichage (vues) et de la gestion des requêtes (contrôleurs).
  • Base de Données : Gestion des données dynamiques (catalogue, utilisateurs) via le dossier BDD.
  • Sécurité : Gestion des sessions PHP pour la partie administration (session_start()).
📁 Structure du projet (MVC)
  • theALaMenthe
    • BDD // Fichiers SQL et base de données
    • config // Configuration (ex: bdd.php)
    • controleurs // Traitement des actions (accueil, carte, auth, admin)
    • images
    • js // Scripts frontend
    • modeles // Gestion des données et requêtes SQL
    • style // Feuilles de style CSS
    • vues // Affichage HTML des pages
    • index.php // Point d'entrée et routeur principal

3. Fonctionnalités Clés

A. Routeur Principal Centralisé

Toutes les requêtes passent par index.php qui agit comme contrôleur frontal. Il sécurise l'application en n'autorisant que certaines pages spécifiques (accueil, carte, auth, admin, mentions) avant d'appeler le bon contrôleur.

B. Espace d'Administration Sécurisé

Mise en place d'un système de session PHP. Une page d'authentification (auth) permet l'accès au panneau d'administration (admin) pour gérer le contenu dynamiquement.

C. Catalogue Dynamique (Carte)

Affichage dynamique des produits (boissons, plats) enregistrés dans la base de données, gérés par les contrôleurs et les modèles.
